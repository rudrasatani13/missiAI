## 2025-03-08 - Hoisting Maps in Loops Resolves Hidden O(N^2) Operations
**Learning:** In Cloudflare Edge deployments handling large KV datasets like `LifeGraph` arrays (potentially containing hundreds of node elements), invoking `Array.find` within `Promise.all` or iteration loops (such as during memory clustering or graph updating in `addOrUpdateNodes` and `constellation-layout.ts`) introduces severe, hidden O(N^2) lookups.
**Action:** When refactoring nested `Array.find` loops using Hash Maps (`Map<K, V>`) for optimization, always initialize and populate the map *outside* of the traversal block, but if the block mutates state (like pushing new `resultNode`s back into the graph array), be sure to dynamically update the map (e.g. `map.set(newKey, newValue)`) inside the loop to avoid missing newly added records and generating duplicates.
